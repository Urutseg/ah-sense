# Contributing

Thanks for helping shape Auction House Sense.

## Product Guardrails

Contributions should preserve the core product posture:

- Contextual assistance, not dashboard takeover
- Evidence and rationale over judgment
- Conservative recommendations over broad coverage
- No automation of Auction House buying, posting, or undercutting

## Issue Guidance

When filing ontology or recommendation issues, include:

- Item names and item IDs, if known
- Retail patch/client version
- Why the items are comparable
- Any reason the comparison may be risky

False positives are treated as higher priority than missed recommendations.

## Pull Request Guidance

Keep pull requests focused. For future implementation work, include a short note describing the user-facing behavior, trust implications, and any performance considerations.
